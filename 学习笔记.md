## 软件架构

```
Application Layer (application/)
  robot_cmd / chassis / gimbal / shoot
      ↓ 发布-订阅消息总线 (message_center)
Modules Layer (modules/)
  PID / EKF / IMU / Motor / Referee / Remote / ...
      ↓
BSP Layer (bsp/)
  CAN / SPI / I2C / UART / PWM / ADC / GPIO / DWT
      ↓
HAL Layer (Drivers/ + Core/)
  CMSIS + STM32H7xx HAL + FreeRTOS + ARM DSP
```

## FreeRTOS 任务

| 任务 | 频率 | 优先级 | 栈(字) | 核心函数 |
|------|------|--------|--------|---------|
| INS Task | 1kHz | AboveNormal | 1024 | BMI088 + EKF 姿态解算 |
| Motor Task | 1kHz | Normal | 512 | DJIMotorControl() + LKMotorControl() |
| Robot Task | ~100Hz | Normal | 1024 | RobotTask(): cmd→gimbal→shoot→chassis |
| Daemon Task | 100Hz | Normal | 128 | 模块看门狗 + 蜂鸣器 |
| UI Task | 200Hz | Normal | 512 | 裁判系统 UI |

## 第一步：初始化流程 — 从上电到 RTOS 启动

### 启动总图

```
芯片上电复位 → main()
  ├─ HAL_Init()
  ├─ SystemClock_Config()       ← 480MHz
  ├─ MX_GPIO/DMA/SPI/CAN/..._Init()  ← 约 20 个外设初始化
  ├─ RobotInit()
  │   ├─ __disable_irq()         ← 关门初始化
  │   ├─ BSPInit()               ← DWT计时 + RTT日志
  │   ├─ RobotCMDInit()          ← 遥控/视觉/Buffer/消息注册
  │   ├─ GimbalInit()            ← INS_Init(IMU) + 2×GM6020
  │   ├─ ShootInit()             ← 2×M3508摩擦轮 + M2006拨弹 + 舵机
  │   ├─ ChassisInit()           ← 4×M3508 + 裁判系统 + 超电 + 功率计
  │   ├─ chassis_power_control_init() ← 功率限制PID
  │   ├─ OSTaskInit()            ← 创建5个FreeRTOS任务
  │   └─ __enable_irq()          ← 开门
  ├─ MX_FREERTOS_Init()
  └─ osKernelStart()             ← RTOS 启动，不再返回
```

### 关键函数

**RobotInit()** — robot.c L24-50：关中断→BSP→应用Init→创建任务→开中断

**RobotTask()** — robot.c L52-65：每周期 RobotCMDTask→GimbalTask→ShootTask→ChassisTask→功率控制（顺序重要，chassis 需要云台偏角数据）

**BSPInit()** — bsp_init.h：DWT_Init(480) + BSPLogInit()。DWT 是 Cortex-M7 硬件计数器，微秒级计时，不依赖中断。

## 专题：Pub-Sub 发布-订阅消息总线

### 本质

全局的、按名字查找的、带队列的 memcpy 中转站。cmd 往 `"chassis_cmd"` 里 memcpy 一个 `Chassis_Ctrl_Cmd_s`，chassis 从 `"chassis_cmd"` 里 memcpy 出来——双方只约定话题名和结构体大小，互不知道对方的存在。

### 核心接口

| 函数 | 作用 |
|------|------|
| `PubRegister(name, data_len)` | 注册发布者 → 创建话题或返回已有话题 |
| `SubRegister(name, data_len)` | 注册订阅者 → 创建订阅者并挂到话题链表 |
| `PubPushMessage(pub, data_ptr)` | 发布 → memcpy 数据到所有订阅者的队列 |
| `SubGetMessage(sub, data_ptr)` | 订阅 → memcpy 从队列取出数据 |

### 全部 8 个话题

| 话题名 | 数据类型 | 发布者 | 订阅者 |
|--------|----------|--------|--------|
| `"gimbal_cmd"` | Gimbal_Ctrl_Cmd_s | robot_cmd | gimbal |
| `"gimbal_feed"` | Gimbal_Upload_Data_s | gimbal | robot_cmd, chassis |
| `"chassis_cmd"` | Chassis_Ctrl_Cmd_s | robot_cmd | chassis |
| `"chassis_feed"` | Chassis_Upload_Data_s | chassis | robot_cmd, gimbal, shoot |
| `"shoot_cmd"` | Shoot_Ctrl_Cmd_s | robot_cmd | shoot |
| `"shoot_feed"` | Shoot_Upload_Data_s | shoot | robot_cmd |
| `"power_cmd"` | Chassis_Power_Data_s | power_ctrl | chassis |
| `"UI_feed"` | Referee_Upload_Data_s | chassis | robot_cmd |

## 第二步：数据流 — 从遥控器到电机转动

### 完整链路

```
DJI DT7 遥控器 (2.4GHz) → DR16 接收机 → UART5 (DBUS, 18字节, ~70Hz)
  ↓ 中断回调 RemoteControlRxCallback()
  ↓ sbus_to_rc() 解析 → rc_ctrl[TEMP]
  ↓ 指针传递
robot_cmd.c RobotCMDTask() @100Hz
  ↓ RemoteControlSet() 摇杆→控制量
  ↓ PubPushMessage() → message_center
chassis.c ChassisTask()
  ↓ SubGetMessage() → chassis_cmd_recv
  ↓ 坐标旋转(云台系→底盘系)
  ↓ MecanumCalculate(): vt = ±vx ± vy - wz × 中心距
  ↓ DJIMotorSetRef(motor, vt)
Motor Task @1kHz
  ↓ DJIMotorControl(): 速度环PID → 电流环PID → CAN帧
  ↓ CANTransmit() → FDCAN1
M3508 电调 → 电机转动
```

### DBUS 帧解析

18 字节解包后填入 `rc_ctrl[TEMP]`：
- `rc.rocker_l1`：左摇杆竖直（ch2 - 1024，约 ±660）
- `rc.rocker_l_`：左摇杆水平（ch3 - 1024）
- `rc.rocker_r_`：右摇杆水平（ch0 - 1024）
- `rc.rocker_r1`：右摇杆竖直（ch1 - 1024）
- `rc.dial`：左上拨轮
- `rc.switch_left/right`：三档开关（1=上, 3=中, 2=下）

### 麦轮运动学

```
        前 ↑
  (lf)──┼──(rf)
   │    │    │
   │    +    │  → 右
  (lb)──────(rb)

轮速 = 平移分量 + 旋转分量（天然叠加，互不干扰）

vt_lf = -vx - vy - wz × LF_CENTER
vt_rf = -vx + vy - wz × RF_CENTER
vt_lb = +vx - vy - wz × LB_CENTER
vt_rb = +vx + vy - wz × RB_CENTER
```

### 底盘模式

| 模式 | 枚举值 | 行为 |
|------|--------|------|
| `CHASSIS_ZERO_FORCE` | 0 | 所有电机掉电，蜂鸣器报警 |
| `CHASSIS_NO_FOLLOW` | 2 | 全向平移，wz 手动控制 |
| `CHASSIS_ROTATE` | 1 | 小陀螺：wz 回中时自动 2000，左摇杆控制平移 |
| `CHASSIS_FOLLOW_GIMBAL_YAW` | 3 | 跟随云台（需 IMU，不可用） |

## 第三步：DM 达妙 2325 电机调试

### DM vs DJI 电机

| 特性 | DJI (M3508/M2006/GM6020) | DM (2325) |
|------|--------------------------|-----------|
| CAN 帧格式 | 一帧带 4 个电机，各取偏移 | 一帧一个电机，点对点 |
| 控制方式 | MCU 计算 PID → 发电流值 | MCU 发 MIT 指令 → 电机内部 PID |
| ID 体系 | tx_id = 电机编号 (1-8) | master_id (MCU→电机) + can_id (电机→MCU) |
| 默认 ID | 标准 DJI ID | master_id=0x00, can_id=0x01 |

### MIT 模式原理

CAN 帧 8 字节，5 个字段共 64 位：

```
Byte:    0        1        2        3        4        5        6        7
     ┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
     │p_des   │p_des   │v_des   │v_lo│kp │  kp    │  kd    │kd_lo│tq│  tq   │
     │[15:8]  │[7:0]   │[11:4]  │[3:0]│[11:8]│[7:0]  │[11:4]  │[3:0]│[11:8]│[7:0]│
     └────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘
      16bit        12bit        12bit       12bit       12bit
   position_des   velocity_des     Kp          Kd        torque_des
```

电机内部计算：`τ = Kp × (pos_des - pos) + Kd × (vel_des - vel) + τ_ff`

### 2325 电机参数

| 参数 | 值 | dmmotor.h 宏 | 备注 |
|------|-----|-------------|------|
| 位置 | ±12.5 rad | `DM_P_MIN/MAX` | 不变 |
| 速度 | ±200 rad/s | `DM_V_MIN/MAX` | 原代码 45，改了 |
| 力矩 | ±10 N·m | `DM_T_MIN/MAX` | 原代码 18，改了 |
| Kp | 0~5 | `DM_KP_MIN/MAX` | 新增 |
| Kd | 0~5 | `DM_KD_MIN/MAX` | 新增 |

### float_to_uint / uint_to_float

CAN 帧传输整数（12/16bit），物理量是浮点，需线性映射：

```
float_to_uint(x, min, max, bits):
  raw = (x - min) × (2^bits - 1) ÷ (max - min)
  例: float_to_uint(10, -200, 200, 12) = 2149 → CAN 帧 0x0865

uint_to_float(raw, min, max, bits):
  x = raw × (max - min) ÷ (2^bits - 1) + min
```

**bits 由 MIT 协议决定**：位置 16bit，其余 12bit。

### MIT 模式调试经验

**问题**：电机 LED 始终红灯（未进入 MIT 模式），硬编码上位机验证过的帧也不工作。

**原因**：
1. `DMMotorInit` 在 `osKernelStart()` 前发 0xFC 使能，`DMMotorTask` 调度器启动后才发控制帧，中间有几百毫秒空窗，电机收不到连续 MIT 帧超时掉线
2. dmmotor.h 范围宏不匹配（V_MAX=45 vs 实际 200，T_MAX=18 vs 实际 10）

**解决**：
1. 修改 dmmotor.h 范围宏
2. 将使能命令从 DMMotorInit 移到 DMMotorTask 开头（L129-130），紧跟控制帧
3. 用 CAN2（空闲，避免干扰底盘 CAN1）
4. float_to_uint 的参数范围全改成电机实际值

### Ozone 调试 DM 通信

- Watch 窗口加 `dm_test`，展开看：
  - `dm_test->motor_can_instace->tx_buff[0..7]` — 发出的 CAN 帧
  - `dm_test->motor_can_instace->rx_buff[0..7]` — 收到的 CAN 帧
  - `dm_test->motor_daemon->temp_count > 0` — 通信成功

## 第四步：底盘方向调试

### 摇杆映射修改

L294-296 最终映射（`rocker_l_→vx`，`rocker_l1→vy` 交换并取反）：

```c
chassis_cmd_recv.vx = -26.0f * (float)rc_data[TEMP].rc.rocker_l_;   // 左水平→前后
chassis_cmd_recv.vy = -26.0f * (float)rc_data[TEMP].rc.rocker_l1;  // 左竖直→左右
chassis_cmd_recv.wz = +15.0f * (float)rc_data[TEMP].rc.rocker_r_;   // 右水平→自转
```

### 麦轮公式调向

`MecanumCalculate()` 改为反向符号（因四个电机全设 MOTOR_DIRECTION_REVERSE）：

```c
vt_lf = +chassis_vx + chassis_vy + chassis_cmd_recv.wz * LF_CENTER;
vt_rf = +chassis_vx - chassis_vy + chassis_cmd_recv.wz * RF_CENTER;
vt_lb = -chassis_vx + chassis_vy + chassis_cmd_recv.wz * LB_CENTER;
vt_rb = -chassis_vx - chassis_vy + chassis_cmd_recv.wz * RB_CENTER;
```

### 右边开关模式

```c
if (switch_is_up(rc_data[TEMP].rc.switch_right))
    chassis_cmd_recv.chassis_mode = CHASSIS_ROTATE;    // 上=小陀螺
else
    chassis_cmd_recv.chassis_mode = CHASSIS_NO_FOLLOW; // 中/下=手动
```

## 第五步：Seasky 协议（视觉通信）

帧结构 = 10 字节头部 + N 字节载荷 + 2 字节 CRC16（CRC-8 头部 + CRC-16 尾部）

**核心消息 `fix_control (0x05)`** — 上位机→MCU，50Hz，21B载荷：

| 字段 | 类型 | 对应变量 |
|------|------|---------|
| yaw | float32 | `vision_recv_data->yaw` |
| pitch | float32 | `vision_recv_data->pitch` |
| fire | uint8 | `vision_recv_data->shoot` |
| vx | float32 | `vision_recv_data->vx` |
| vy | float32 | `vision_recv_data->vy` |
| spin | float32 | `vision_recv_data->spin` |

**关键回传 `sentry_state (0x20)`** — MCU→上位机，≥5Hz，必须严格 20 字节。
